# ProScan — Website

Marketing site (and, after the revamp, web app) for **ProScan**, the Amazon
competitor-research Chrome extension.

- **Sibling repo (the extension/engine):** `../AmazonSellerScraper` (ProScan MV3 extension + optional FastAPI/RAG server)
- **Chrome Web Store listing:** https://chromewebstore.google.com/detail/proscan-amazon-product-sc/bikgignfnljpbmchlemkbbpboigodgap
- **Hosting:** Firebase Hosting — project id `proscanbot` (see `.firebaserc`)

## Current state (Phase 0 complete — Vite build pipeline)

Still the same static marketing landing page, but now built with **Vite** so there's a
clean source → build split for the revamp to grow into. The landing page output is
byte-for-byte the same site as before.

```
proscan-web/
├── index.html         # Vite entry — the landing page markup
├── src/
│   ├── style.css      # All styles (design tokens: #2a343e dark / #fcfcfc light / #5a646e muted)
│   └── main.js        # Hero scan-mockup animation + smooth-scroll (imports style.css)
├── public/            # Copied verbatim into dist/ on build
│   ├── icon128.png    # Logo
│   ├── 404.html       # Branded 404 (Proscan styling)
│   └── googlec3...html # Google Search Console verification
├── dist/              # ⚠️ BUILD OUTPUT ONLY (git-ignored) — regenerated by `npm run build`
├── vite.config.js
├── package.json       # type: module; vite dev/build/preview/deploy scripts
├── firebase.json      # hosting.public = "dist"
└── .firebaserc        # default project: proscanbot
```

> The old footgun (hand-written source living in `dist/`) is gone — `dist/` is now
> generated build output. Edit `index.html`, `src/`, and `public/` only.

## Develop & deploy

```bash
npm install            # one-time
npm run dev            # local dev server with HMR
npm run build          # build to dist/
npm run preview        # serve the built dist/ locally
npm run deploy         # build + firebase deploy --only hosting (project: proscanbot)
```

## Revamp

Adds sign-in, saving client/competitor data, and integration with the ProScan
extension so the site can surface live scrape-session data per signed-in user.
See **`CONTEXT.md`** (current-state reference) and **`REVAMP_PLAN.md`** (architecture + phased plan)
— both generated from a research pass over this repo and the extension.
